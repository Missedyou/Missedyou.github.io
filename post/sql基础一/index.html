1. 数据库、表的创建、定义与修改

数据库的创建与删除

create table `mytable`; //创建一个名为mytable的数据库

drop table `mytable`; //删除一个名为mytable的数据库

表的创建

create table `table_name`; //不定义具体表结构

create table `table_name`(
    `id` int(11) not null auto_increment, //非空, 自增
    `name` varchar(255) not null,
    primary key(`id`) //定义主键
)default charset=utf8; // 声明编码方式

修改表结构

alter table my_table add (my_column int(11)); //新增字段
alter table my_table rename column my_column to new_column_name; //修改字段名称
alter table my_table modify my_column float(3, 2); //修改字段属性
alter table my_table drop column my_column; //删除字段

2. 约束

2.1 主键约束

主键是表记录的唯一标识, 不能重复、不可为空即: unique + not null。 主键通常是一个字段, 也可以是多个字段复核而成。

2.2 外键约束

外键是为了确保表与表之间引用的完整性, 一张表中的外键对应另一张表中的主键, 外键是可重复的也可为空。(外键会影响insert的速度会造成性能问题, 因此在实际业务中几乎不会使用外键, 外键的功能通常在业务代码中实现)

2.3 唯一性、非空、默认值约束

除了对主键有unique、not null约束之外, 也可以对其他字段进行unique、not null定义, 同时可以对字段给与默认值即default 0.00。

3. 设计原则

数据库的设计原则源于E-R图(Entity Relationship Diagram), 表数量越少、字段越少越易于操作。
